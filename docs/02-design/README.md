This section defines how Clada OS should feel and behave for contractor-led businesses.

## Design Aim

Clada OS should feel professional, efficient, and trustworthy. Contractor workflows should be clear, scannable, and operationally useful. Avoid visual choices that make the product feel like a marketing page when the user is trying to perform work.

## Documents

- [DESIGN_PRINCIPLES.md](DESIGN_PRINCIPLES.md): design rules for product interfaces and workflow surfaces.

## Design Review Questions

1. Can the user understand the current state quickly?
2. Is the next action obvious?
3. Does the interface respect business context and customer data sensitivity?
4. Does the design support repeated operational use?
5. Are module-specific terms separated from platform-level language?
