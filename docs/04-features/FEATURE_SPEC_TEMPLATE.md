# Feature Specification Template

| Field | Value |
| --- | --- |
| Document ID | FEAT-template |
| Status | Draft |
| Owner | TBD |
| Review cycle | Per feature milestone |
| Last reviewed | YYYY-MM-DD |

## Summary

State the feature in one or two paragraphs.

## Problem

What customer or operational problem does this solve?

## Evidence

List research, customer signals, support issues, analytics, or operational observations.

## Product Scope

Define what is in scope and out of scope.

## Platform Classification

Choose one:

- SolarGRANT Pro module feature
- Clada OS platform capability
- module feature with platform extraction candidate
- experiment

Explain why.

## User Workflow

Describe the expected workflow and key states.

## Design Requirements

Describe interface, content, accessibility, and operational clarity requirements.

## Architecture Notes

Describe data model, API, integration, AI, automation, and security implications.

## Risks

List product, technical, operational, security, privacy, and compliance risks.

## Verification Plan

Define tests, manual checks, migration checks, and release verification.

## Rollout Plan

Describe sequencing, flags, migration, support needs, and rollback or mitigation.

## Documentation Updates

List documents that must be updated before release.
