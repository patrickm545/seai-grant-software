# Clada OS Design System

## Design direction

Clada OS should feel like trusted Irish business software: calm, clear, reliable, and operationally useful.

Avoid overdesigned SaaS trends, excessive gradients, novelty AI visuals, and crowded dashboards.

## Visual principles

- Clear hierarchy
- Strong readability
- Spacious layouts
- Practical dashboards
- Consistent cards and tables
- Obvious primary actions
- Minimal visual noise

## Interface personality

The interface should feel established, trustworthy, professional, useful, calm, and fast to understand.

It should not feel overly playful, generic, cluttered, or experimental.

## Layout rules

- Keep primary actions visible where possible.
- Use cards for grouped business information.
- Use tables for operational lists.
- Use detail pages for deeper records.
- Do not bury important workflow actions.
- Empty states should explain the next action.

## Forms

Forms should be short, grouped, and easy for non-technical users to complete.

Avoid asking homeowners for information the installer can collect later unless it clearly improves lead quality.

## Dashboards

Dashboards should show what needs attention, not decorative metrics.

Useful dashboard content includes new leads, follow-ups due, quotes waiting, missing documents, jobs in progress, and recent activity.

## Copy rules

Use plain English. Avoid jargon unless the installer would naturally use it.
