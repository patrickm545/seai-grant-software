## Main UX goal

Users should always know what needs attention, what has changed, what the next action is, and where each record stands.

## Customer-facing UX

Customer-facing forms should be simple, reassuring, and quick.

The customer should understand why information is needed and what will happen next.

## Business user UX

Business users should be able to scan the system quickly and act without confusion.

Important records should show contact details, property details, lead quality, workflow information, notes, and next action in one place.

## Admin UX

Admin workflows should reduce repeated manual work.

Where possible, support copyable summaries, clear statuses, templates, and structured notes.

## UX rules for new features

- Do not add a feature unless the next action is clear.
- Do not add complex settings before the core workflow works.
- Do not make forms feel like paperwork.
- Do not hide sales-critical information.
- Keep status labels consistent across the app.
