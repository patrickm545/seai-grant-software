## Objective

Create the product and engineering foundation for Clada OS before major feature development continues.

## Business reason

The repository is moving from a SolarGRANT Pro grant checker toward Clada OS, a broader business platform. The repository needs shared documents that Codex and future developers can read before making changes.

## Scope

- Maintain the existing working product.
- Establish documentation as the source of truth.
- Define product direction.
- Define design direction.
- Define engineering rules.
- Define future sprint format.

## Out of scope

- Major UI rebuilds.
- Database redesigns.
- New CRM features.
- New quote features.
- New AI features.

## Required documents

- Company vision
- Product positioning
- Clada OS vision
- Current SolarGRANT Pro state
- Roadmap
- Feature decision framework
- Design system
- UI and UX principles
- Architecture notes
- Security and GDPR notes
- Sprint template
- First decision records

## Definition of Done

Sprint 0 is complete when the repository contains enough context for Codex to safely begin Sprint 1.1 without relying on long chat history.
